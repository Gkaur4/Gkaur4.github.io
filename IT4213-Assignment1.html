2) a) In general a viewport is the part of a webpage that the user can see. On mobile devices, there are two types of viewports, one is the layout viewport. The layout viewport is how the website is rendering the website. The other is the visual viewport which is whats actually being shown on the screen and is the pixels the device's screen in rendering. So, the visual viewport is whats visible on-screen meanwhile the kayout viewport covers the elements on a page. For example, if the user was to zoom into the screen, the visual viewport shrinks but the layout viewport stays the same.

b) On this IT 4213 demo site, the text appears so small on a mobile device because it does not have the mega tag. Infact, Apple had introduced the "viewport meta tag" so web developers can control the viewport's size and scale. If the mobile site has something like the following then the page and text will appear like normal:
meta name = "viewport" content = "width=device-width, initial-scale=1"
